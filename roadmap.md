# Notch Todo Roadmap

本文档记录后续开发路线。当前版本已经满足主要日常需求，后续目标是提升可靠性和少量日常便利，避免扩展成完整任务管理器。

## 原则

- Markdown 文件继续作为唯一数据源。
- 应用只负责显示任务和切换 checkbox。
- 不新增任务管理系统能力。
- 每个阶段都应小步实现、单独验证。
- 发现真实使用问题优先于新增功能。

## Phase 1：稳定性维护

目标：让当前版本更耐用，不增加新工作流。

### 1. 错误状态细分 ✅

区分以下错误：

- 文件不存在
- 权限失效
- Markdown 格式错误
- 写入冲突

验收标准：

- 不同错误显示不同短标题和恢复动作。
- 错误文案保持简短，不暴露不必要的底层细节。
- 现有文件选择恢复入口仍然可用。

### 2. 写入冲突恢复 ✅

当任务文件被外部修改导致 toggle 失败时，提供“重新加载”按钮。

验收标准：

- 不覆盖外部修改。
- 用户可以一键重新读取最新文件内容。
- 失败状态不会阻塞重新选择文件。

### 3. 文件授权恢复 ✅

当 security-scoped bookmark 失效时，直接引导重新选择文件。

验收标准：

- 重启后权限失效不会卡在不可操作状态。
- 重新选择文件后任务列表恢复正常。
- 不增加额外权限配置入口。

## Phase 2：日常便利

目标：减少手动找文件、重启应用的摩擦。

### 4. 设置页增加“打开 Markdown 文件” ✅

使用默认编辑器打开当前任务文件。

验收标准：

- 按钮只在已选择文件时显示。
- 文件不存在时进入现有错误状态。
- 不内置编辑器，不修改任务文本。

### 5. 设置页增加“在 Finder 中显示” ✅

快速定位当前任务文件。

验收标准：

- 文件存在时可在 Finder 中定位。
- 文件丢失时显示明确错误。
- 不改变当前选择的文件。

### 6. 设置页增加“重新加载任务” ✅

手动刷新当前 Markdown 文件。

验收标准：

- 点击后重新读取文件。
- 保留当前面板状态。
- 读取失败时进入对应错误状态。

## Phase 3：显示策略

目标：让它更安静，但不加复杂偏好系统。

### 7. 只在有未完成任务时显示 ✅

全部完成后，紧凑面板可以自动变得更低调，或隐藏到下次文件变化。

验收标准：

- 不影响文件监听。
- 文件变化后能重新出现。
- 不引入复杂显示规则。

### 8. 紧凑状态点击展开 ✅

在 hover 之外提供一个明确交互方式。

验收标准：

- 点击展开与 hover、pin 逻辑不冲突。
- 外部点击收起行为保持一致。
- 不扩大不可见点击区域到容易误触的程度。

### 9. 长列表 scroll fade

展开面板内容可滚动时，在顶部或底部加非常淡的边缘提示。

验收标准：

- 不增加视觉噪声。
- 不影响任务点击和滚动。
- Reduce Motion 设置不受影响。

## Phase 4：分发与维护

目标：方便长期自用和迁移机器。

### 10. 安装脚本增加版本检查

显示当前安装版本和即将安装版本。

验收标准：

- 仍保持一条命令安装。
- 没有安装旧版本时正常处理。
- 不加入自动下载或自动更新。

### 11. 增加导出诊断信息

导出非敏感诊断信息，例如 app 版本、选中文件路径、权限状态和解析状态。

验收标准：

- 不导出 Markdown 文件内容。
- 不导出密钥、凭证或系统隐私数据。
- 输出格式便于复制到 issue 或交接文档。

### 12. Developer ID 签名和公证说明

只补充文档，说明本地 ad-hoc 签名和正式分发的区别。

验收标准：

- README 说明当前本地安装方式。
- README 说明如果要分发给他人，需要 Developer ID 签名和公证。
- 不把正式分发作为当前必须完成项。

## 暂不纳入

以下能力暂不做：

- 新增、删除、编辑任务
- 多文件或多项目
- 日期自动切换
- 提醒、通知、番茄钟
- 第三方任务服务集成
- 复杂主题或偏好系统
- 云同步
- 外接屏支持

## 建议的下一个实现包

优先评审并实现以下三项：

1. 错误状态细分
2. 设置页增加“重新加载任务”
3. 设置页增加“打开 Markdown 文件”

这三项对日常使用最有帮助，且不会改变应用的核心边界。
